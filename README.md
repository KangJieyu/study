## 开发规范

1. 接口中不需要 `public` 和 `abstract` 关键字
2. 方法命名规则除小驼峰式命名外，还需遵循：
   1. 获取单个对象用 `get` 前缀
   2. 获取多个对象用 `list` 前缀
   3. 获取统计数量用 `count` 前缀
   4. 插入数据方法用 `save` 或 `insert` 前缀
   5. 删除数据方法用 `remove` 或 `delete` 前缀
   6. 更新数据方法用 `update` 前缀
3. 领域模型命名规范：
   1. 数据对象：xxxDO，xxx为数据表名
   2. 数据传输对象：xxxDTO，xxx为业务领域名称
   3. 显示对象：xxxVO，xxx为页面名称
